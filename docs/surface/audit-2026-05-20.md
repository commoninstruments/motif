# Surface Audit: Motif

Date: 2026-05-20

## Detected Stack

- TypeScript pnpm/turbo monorepo with workspace packages in `apps/cli`, `packages/motif-sdk`, `packages/motif-mcp`, and `packages/motif-server`.
- Public package surfaces: `@howells/motif-cli`, `@howells/motif-sdk`, `@howells/motif-mcp`, and deprecated `@howells/motif-server`.
- Runtime surfaces: Node 22 CLI, Node SDK using platform `fetch`, local stdio MCP server, terminal Ink Studio.
- Agent surfaces present: CLI structured output, stdin JSON, `--dry-run`, `--describe`, field masks, MCP tools/resources, package smoke tests, CLI contract tests, root `AGENTS.md`, `llms.txt`, `.mcp.json`, and package-scoped `apps/cli/AGENTS.md`.
- No HTTP API, hosted web app, public OpenAPI spec, remote MCP transport, retrieval/RAG corpus, or multi-agent runtime in this public repo.

### 3. CLI Error Semantics Stop Short Of Full Autonomy

The CLI has strong structured error objects, but every terminal error path still collapses to generic process failure semantics.

| Severity | File | Issue | Dimension |
| --- | --- | --- | --- |
| Medium | `apps/cli/src/utils/errors.ts`, `apps/cli/src/utils/output.ts` | Structured error body includes status/doc_uri/suggestions, but process exit codes are not semantic. | CLI Design, Error Handling |
| Medium | `apps/cli/src/utils/output.ts` | No `trace_id` or `instance` field for correlating failed CLI runs. | Error Handling |
| Low | `apps/cli/src/commands/describe.ts` | Error docs are local schema entries, not linked to stable markdown docs. | Error Handling, Discovery |

Suggested approach: map error metadata to sysexits-style exit codes, add generated trace IDs to structured errors, and document error code pages under `docs/errors/`.

Dependencies: decide whether changing exit codes is a breaking CLI contract.

### 5. Auth Is Pragmatic And Documented, But Broad

Motif delegates auth to fal’s API key model, which is workable for local tools but not fine-grained. The local trust boundaries are now documented.

| Severity | File | Issue | Dimension |
| --- | --- | --- | --- |
| Resolved | `docs/security.md`, `README.md`, `packages/motif-mcp/README.md` | Auth docs now spell out `FAL_KEY`, local config, MCP trust, history exposure, and `--ephemeral` caveats. | Authentication |
| Low | `packages/motif-mcp/src/index.ts` | MCP server exits with a plain fatal message when `FAL_KEY` is missing. | Authentication, Error Handling |

Suggested approach: keep `docs/security.md` current as fal auth capabilities change, and improve the MCP startup error shape if this becomes a support issue.

Dependencies: fal’s available key/scoping model.

### 6. Testing Is Strong For Deterministic Surfaces, Missing Evals

The deterministic test suite is unusually good for a CLI/MCP package, but there is no eval or trace regression layer.

| Severity | File | Issue | Dimension |

| Low | `.github/workflows/ci.yml` | CI runs deterministic tests but no optional eval/canary reporting job. | Testing |
